VSA-PathMoE Documentation

A reproducible test of request-level VSA routing over very small tensor experts, compared with official TinyStories checkpoints and matched dense controls.

Protocol implemented; final DGX and commodity-CPU result pending.

The proposition under test

VSA-PathMoE keeps a compact Transformer backbone resident and selects one narrow feed-forward micro-expert per request. A deterministic Vector Symbolic Architecture router reads only the causal conditioning prefix, assigns one of 64 routes, and holds that route for the continuation. The implementation therefore separates inexpensive symbolic coordination from the tensor path used to predict tokens.

The repository does not yet demonstrate that this design beats a tensor-only language model. It supplies a fail-closed experiment that can establish or reject the narrower TinyStories claim. A positive statement requires quality non-inferiority to the official 8M checkpoint, robust CPU speed and resident-memory advantages, real route and expert contribution, dense-control evidence, and a completed blind review.

Training and quality evaluation are scheduled on NVIDIA DGX Spark with CUDA BF16. Deployment timing and memory are separate CPU FP32 experiments on a commodity x86_64 host at 1, 4, and 8 threads. This split prevents the accelerator used to train the custom model from being confused with the hardware named by the CPU-efficiency claim.

Repository structure and authority

  • config/ — base contract and frozen runtime profiles
  • python/vsa_bench/ — data, router, model, training, evaluation, and benchmark
  • scripts/ — orchestration, reporting, blind review, and verification
  • tests/ — Node.js and Python acceptance and rejection tests
  • docs/ — this maintained long-form technical site
  • docs/specs/ — authoritative Design Specifications
  • .agents/skills/ — imported agent tooling, kept outside the host documentation surface

The versioned Design Specifications are the documented source of truth. DS001 governs coding style, artifact compatibility, source layout, and modular testing. A source change that alters behavior, interfaces, architecture, workflow, or constraints must update the affected specification and HTML chapter together.

The local skills under .agents/skills/ assist repository work but are not product runtime dependencies. Their portable instructions and examples remain inside their own folders. AGENTS.md carries the synchronized skill catalog and future-agent reading order.

Imported skill catalog

The current agent-tooling catalog is achilles-specs, antropic-skill-build, article-build, create-akus, cskill-build, dgskill-build, gamp-specs, manage-ploinky-agents, oskill-build, and review-specs. These skills support project bootstrap, specification review, portable skill forms, article generation, knowledge-unit construction, and agent operations.

This repository consumes them downstream, so their detailed contracts remain in their own SKILL.md files rather than becoming host-project HTML chapters or DS specifications. When that folder set changes, update AGENTS.md, this catalog statement, and the generated specification matrix in the same change.

TinyStories is appropriate because its authors specifically study coherent language generation below ten million parameters. That scope is also a limitation: even a successful result remains a TinyStories result, not evidence of general language-model superiority. See the primary sources and comparison protocol.

Verification model

Tests exercise both success and rejection paths for prepared-data identity, checkpoint compatibility, model-list provenance, workload manifests, blind packets, benchmark runtime selection, and report gates. Offline smoke proves plumbing only. Full scientific artifacts must come from a clean, frozen run and are validated again when the report is assembled.

The documentation itself is served and checked through the project-local verification scripts. Because headers, footers, and specifications are loaded with fetch(), open this site through HTTP rather than directly from a file: URL.
