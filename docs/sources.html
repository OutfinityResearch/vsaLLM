Primary evidence and related work

Sources and claim boundaries

These sources justify dataset roles, baseline identity, operational choices, and related-work positioning. They do not supply the missing experimental result.

TinyStories data and checkpoints

  1. Ronen Eldan and Yuanzhi Li, “TinyStories: How Small Can Language Models Be and Still Speak Coherent English?” The primary paper motivates a synthetic domain in which sub-10M models can be evaluated for coherent generation. It supports the benchmark domain, not a VSA result.
  2. Official TinyStories dataset repository, including TinyStories-train.txt, TinyStories-valid.txt, and the published prompt file. The DGX profile pins resolved revision f54c09fd…f9c64; the execution manifest must retain the full SHA.
  3. Official model repositories for TinyStories-8M and TinyStories-33M. The DGX profile pins revisions 8612e3b1…30af and 2ad0a164…fc8, respectively.
  4. The checkpoint author's context-length clarification distinguishes the 512-token training window from 2,048 stored positional embeddings. The protocol records both values rather than treating artifact capacity as training evidence.
  5. Rowan Zellers et al., “HellaSwag: Can a Machine Really Finish Your Sentence?”, and the published dataset. HellaSwag is a secondary diagnostic, not the TinyStories primary criterion.

Mixture-of-Experts and deterministic access

  1. Noam Shazeer et al., “Outrageously Large Neural Networks: The Sparsely-Gated Mixture-of-Experts Layer.” This establishes the learned sparse-gating lineage.
  2. William Fedus, Barret Zoph, and Noam Shazeer, “Switch Transformers: Scaling to Trillion Parameter Models with Simple and Efficient Sparsity.” Switch routing provides a central token-level sparse-MoE comparison and documents BF16 sparse training at a very different scale.
  3. Stephen Roller et al., “Hash Layers for Large Sparse Models.” Deterministic hashing demonstrates that learned gates are not the only sparse-routing design; its token-level hash is not the request-prefix VSA router implemented here.
  4. Guillaume Lample et al., “Large Memory Layers with Product Keys.” Product-key sparse access is relevant to the two-codebook route construction.

These works mean the repository should describe an implemented design distinction rather than assert unqualified novelty. Priority requires a broader literature review; performance requires this repository's controlled experiment.

Vector Symbolic Architectures

  1. Pentti Kanerva, “Hyperdimensional Computing: An Introduction to Computing in Distributed Representation with High-Dimensional Random Vectors.” This is a foundational account of high-dimensional distributed representation and bipolar operations.
  2. Denis Kleyko et al., “A Survey on Hyperdimensional Computing aka Vector Symbolic Architectures, Part I.” The survey situates bundling, binding, and distributed representations across VSA families.

The literature supports the operation family. It does not establish that this byte code, prefix boundary, prototype fitting method, or route assignment captures semantic relevance. Route entropy and ablation losses provide implementation-specific evidence; they are not supplied by citation.

Runtime and DGX Spark

  1. NVIDIA DGX Spark hardware overview, software guide, and container-runtime guide. These document the supplied 240 W power requirement, supported software stack, and GPU exposure through the NVIDIA Container Runtime.
  2. NVIDIA's NGC guidance for DGX Spark. It covers registry authentication, ARM64 tooling, GPU container execution, persistent volumes, and version pinning.
  3. NVIDIA NGC PyTorch 25.11 container entry. This is the human-readable base selected by the Dockerfile; the official run additionally records the registry RepoDigest resolved at pull time.
  4. Docker image-digest documentation. A digest identifies immutable image content, whereas a tag can move.
  5. PyTorch scaled dot-product attention, automatic mixed precision, and inference mode document the tensor primitives used by the implementation.

Operational documentation changes over time. The frozen run archives exact observed software, driver, CUDA, PyTorch, tag, RepoDigest, derived image ID, and package freeze rather than relying on the latest web page after execution.

What the sources do not prove

No citation demonstrates that this checkpoint is faster, smaller in resident memory, or non-inferior in quality. No parameter-count calculation substitutes for measured process memory. No TinyStories outcome establishes general natural-language ability. No VSA reference establishes that the router selected the most relevant expert.

The defensible contribution at the current stage is a concrete request-level VSA-routed MoE hypothesis and a reproducible, fail-closed comparison protocol. The performance contribution remains pending until its artifacts exist.
